# ICE = Impact × Confidence × Ease (произведение)

> Реализация: `ICE_CONFIG` и `iceScore`/`iceBucket` в `code/shared/src/`. ADR — `decisions/005-ice-product-vs-mean.md`.

Каждый фактор 1–10. Итог — **произведение** (диапазон 1–1000), не среднее.

## Почему произведение, а не среднее

- I=10, C=2, E=10 → mean = 7.3 («средне-высоко», вводит в заблуждение), **product = 200** (правильно
  сигналит: низкая уверенность тянет вниз).
- I=8, C=7, E=6 → product = 336.
- I=8, C=5, E=7 → product = 280.

Произведение **наказывает однобокие гипотезы** — это нужное поведение для приоритизации.

## Пороги (плашки приоритета)

| Диапазон | Бакет              | Смысл            |
| -------- | ------------------ | ---------------- |
| 1–125    | `low` (серый)      | низкий приоритет |
| 126–342  | `medium` (жёлтый)  | средний          |
| 343–729  | `high` (оранжевый) | высокий          |
| 730–1000 | `top` (красный)    | top priority     |

## Anchor-описания (для калибровки)

- **Impact 10** — если подтверждена, гарантированно +≥30 билетов до старта; **5** — +10 при удачном раскладе.
- **Confidence 10** — есть прямые данные/интервью; **5** — аналогии и логика, но не проверено.
- **Ease 10** — полдня без согласований; **5** — неделя + согласование с продюсером.

У каждого фактора в UI обязателен **rationale в одну строку** (иначе Save заблокирован).
